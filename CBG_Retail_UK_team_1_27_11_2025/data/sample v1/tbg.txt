TROUBLESHOOTING GUIDELINES FOR INCIDENTS:

INC-10000 Troubleshooting Guideline:
Intermittent search timeouts occur when cache misses align with heavy index refresh load. Thread pool saturation and stale cache entries lead to NullPointerExceptions in the request handler. Start by checking thread pool saturation, reviewing queue size, active threads, and overall CPU. Validate search cache TTL and check if older objects remain due to partial invalidation. Inspect index refresh logs to confirm if refresh jobs run in parallel on multiple nodes. Enable request-level timeouts to upstream index calls and add circuit breakers to prevent back-pressure. Flush local search cache, restart service gracefully, and scale replicas if demand remains high. Verify by ensuring p95 latency stabilizes under expected threshold and cache-hit ratio exceeds 90%.

INC-10001 Troubleshooting Guideline:
Inventory API timeouts and DB handshake failures usually indicate token-expiry churn or misconfigured connection pools. Review DB connection pool metrics and confirm if maxWaitExceeded logs appear. Validate token TTL differences between environments and ensure env-level config consistency. Implement a token cache with a short TTL to reduce repetitive AuthClient token fetch calls. Increase DB max connections gradually and restart the inventory-service after applying changes. Confirm resolution by comparing error rates, ensuring DB wait times drop, and validating smooth 24-hour operation.

INC-10002 Troubleshooting Guideline:
Stale reads and duplicate allocations stem from race conditions in the cache invalidation handler. Review race logs and inspect invalidation callback behavior. Apply a compare-and-swap mechanism to enforce atomic invalidation. Perform a cluster-wide invalidation flush using version tokens. Rebuild and backfill inventory snapshots to correct stale entries. Add concurrency-focused unit tests to validate multi-threaded invalidation logic. Verify resolution by ensuring no stale reads for 48 hours and consistent allocation results.

INC-10003 Troubleshooting Guideline:
Elevated inventory p95 latency during reindex indicates full table scans and insufficient DB read replicas. Examine SQL slow-query logs and identify queries missing proper indexes. Add required indexes, route heavy read traffic to new read replicas, and shift reindex operations to off-peak hours. Enable query sampling to catch abnormal query patterns early. Validate by keeping CPU under 65% and ensuring p95 remains below 300ms even during maintenance jobs.

Implement idempotency keys to avoid duplicate processing. Restart broker nodes and monitor queue lag. Confirm stability once DLQ growth stops and reconciliation reports zero missing payments.
